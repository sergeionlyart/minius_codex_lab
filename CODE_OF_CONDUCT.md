# Code of Conduct

## Our commitment

Project participants commit to a respectful, inclusive, evidence-oriented
environment. Differences in legal systems, languages, professional roles,
experience, and technical judgment should be handled with care and good faith.

## Expected behavior

- Be respectful, specific, and constructive.
- Critique ideas and evidence, not people.
- State uncertainty and correct mistakes transparently.
- Respect privacy, confidentiality, and intellectual property.
- Use synthetic examples in public discussions.
- Accept maintainer decisions made to protect users, contributors, or the
  public data boundary.

## Unacceptable behavior

- Harassment, threats, discriminatory language, sexualized attention, or
  personal attacks.
- Publishing another person's private information without permission.
- Uploading real legal matters, personal data, restricted information,
  credentials, or exploit details to public issues or pull requests.
- Knowingly misrepresenting legal authority, project affiliation, test results,
  release status, or security findings.
- Disrupting project infrastructure or repeatedly ignoring maintainer safety
  instructions.

## Scope

This policy applies in repository discussions, issues, pull requests, reviews,
project events, and public representation of the project.

## Reporting

Report ordinary conduct concerns privately to a repository maintainer using a
private GitHub channel available from the verified repository account. Do not
put sensitive evidence in a public issue. Security vulnerabilities follow
`SECURITY.md`.

No private contact address is invented in this initial release. If no suitable
private channel is available, retain the sensitive details and ask a
maintainer publicly only for a private contact method.

## Enforcement

Maintainers may clarify expectations, edit or remove content, close
interactions, issue warnings, temporarily restrict participation, or
permanently ban participation. Decisions should be proportionate, documented
privately where appropriate, and protect the reporter's privacy.

Retaliation against a good-faith reporter is prohibited. Knowingly false or
malicious reports may themselves violate this policy.
