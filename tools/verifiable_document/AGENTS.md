# AGENTS.md — инструмент проверяемого документа

Эти правила дополняют корневой `AGENTS.md` для `tools/verifiable_document/`.

- Инструмент должен быть детерминированным, локальным и без сетевых вызовов.
- Не выполняй OCR автоматически и не маркируй machine-extracted текст как human-verified.
- Не ослабляй проверки уникальности IDs, разрешения ссылок и SHA-256 ради успешной сборки.
- HTML — канонический выход для машинной проверки. DOCX — производный формат с internal bookmarks/hyperlinks и обязательной визуальной проверкой человеком.
- Изменение схемы требует обратной совместимости либо новой `schema_version`, тестов и записи решения в `memory/decisions/`.
- Добавляй воспроизводимый `unittest` для каждого исправленного дефекта.
- Не включай реальные закрытые материалы в fixtures; используемые примеры должны быть явно учебными.
