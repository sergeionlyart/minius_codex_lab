# ADR-0001: Repository memory

- **Status:** accepted
- **Date:** 2026-07-11

## Context

Параллельные исследовательские чаты должны разделять устойчивое состояние без зависимости от скрытой памяти модели и без обязательной внешней БД.

## Decision

Использовать Git как журнал контрольных точек, `memory/CURRENT.md` как небольшой оперативный снимок, `memory/sessions/` как handoff и `matters/` как фактическое досье.

## Consequences

- Состояние прозрачно, diff-able и проверяемо.
- Требуется дисциплина обновления и merge.
- Сырые transcripts не становятся памятью автоматически.
- Позднее слой можно индексировать Obsidian/SQLite/RAG без изменения предметных форматов.
