# Операционная память проекта

`memory/` — небольшой, отслеживаемый Git слой для передачи устойчивого состояния между параллельными Codex-сессиями. Это не архив рассуждений и не склад сырого чата.

## Файлы

- `CURRENT.md` — актуальный снимок: цели, active matters, решения, риски, ветки, следующие действия. SessionStart hook читает его автоматически.
- `OPEN_QUESTIONS.md` — открытые вопросы, владельцы и критерии закрытия.
- `DECISIONS.md` + `decisions/` — индекс и ADR-подобные устойчивые решения.
- `GLOSSARY.md` — согласованные термины/сокращения.
- `SOURCE_POLICY.md` — источники, даты актуальности и правила provenance.
- `sessions/` — один handoff-файл на исследовательскую сессию.
- `index.yaml` — небольшой machine-readable индекс.

## Что сюда не помещать

- секреты, токены, пароли;
- персональные/ограниченные данные;
- полные source files;
- сырые tool logs и chat transcripts;
- скрытые chain-of-thought или подробные рассуждения;
- утверждения без ссылки на matter/evidence/commit.

## Правило обновления

Обновляй `CURRENT.md` только при изменении устойчивого состояния. Временные наблюдения держи в session note или research file. Каждое существенное изменение памяти должно иметь локальный `memory:` или `decision:` commit.
