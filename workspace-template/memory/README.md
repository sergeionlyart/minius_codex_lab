# Операционная память проекта

`memory/` — небольшой прозрачный слой для передачи устойчивого состояния между
параллельными Codex-сессиями. Он становится отслеживаемым Git-слоем только в
режимах `local-git` и `private-approved`; безопасный default `untracked`
оставляет изменяемые файлы локальными и ignored. Режим фиксирует
`scripts/init_workspace.py` в локальном `.git/config`. Это не архив рассуждений
и не склад сырого чата.

## Режимы хранения

- `untracked` — `CURRENT.md`, индексы, session notes и реальные matters не
  входят в Git; remote не создаётся.
- `local-git` — эти файлы можно явно stage/commit в локальный repository после
  проверки классификации.
- `private-approved` — такое же tracking-поведение после организационного
  решения; private remote подключается отдельно и не разрешает запрещённые
  классы данных автоматически.

Проверьте выбранный режим:

```bash
git config --local --get minius.memoryMode
```

## Файлы

- `CURRENT.md` — актуальный снимок: цели, active matters, решения, риски, ветки, следующие действия. SessionStart hook читает его автоматически.
- `OPEN_QUESTIONS.md` — открытые вопросы, владельцы и критерии закрытия.
- `DECISIONS.md` + `decisions/` — индекс и ADR-подобные устойчивые решения.
- `GLOSSARY.md` — согласованные термины/сокращения.
- `SOURCE_POLICY.md` — источники, даты актуальности и правила provenance.
- `sessions/` — один handoff-файл на исследовательскую сессию.
- `index.yaml` — небольшой machine-readable индекс.

## Что сюда не помещать

- секреты, токены, пароли;
- персональные/ограниченные данные;
- полные source files;
- сырые tool logs и chat transcripts;
- скрытые chain-of-thought или подробные рассуждения;
- утверждения без ссылки на matter/evidence/commit.

## Правило обновления

Обновляй `CURRENT.md` только при изменении устойчивого состояния. Временные
наблюдения держи в session note или research file. В tracked-режиме каждое
существенное изменение памяти должно иметь локальный `memory:` или `decision:`
commit после явного stage. В `untracked` handoff остаётся локальным; скрипты не
выполняют commit или push автоматически.
