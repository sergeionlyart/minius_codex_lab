# Support

## Where to start

1. Read `README.md` and the documentation under `docs/`.
2. For an installed workspace, read its own `README.md`, `AGENTS.md`, and
   `SECURITY.md`.
3. Search existing issues.
4. Use the relevant issue form with a minimal synthetic reproduction.

GitHub Discussions may be used for non-sensitive questions if it is enabled on
the verified repository.

## Supported topics

- installation and checksum verification;
- reproducible builds and validation failures;
- skill/role contract defects;
- synthetic fixture and document-tool behavior;
- compatibility and documentation gaps.

## Not provided

- legal advice, representation, or approval of legal documents;
- review of a real matter through public issues;
- recovery of secrets or private user data;
- guaranteed response or remediation times;
- support for modified distributions that cannot reproduce the issue from
  public source.

For vulnerabilities or data exposure, use the private process in
`SECURITY.md`, not an issue.

When asking for help, include version, operating system, Python version, exact
command, sanitized output, and reproduction steps. Remove local usernames,
paths, document contents, tokens, and identifiers.
